Description

MongooseDB, The WeatherExpress project, uses Cloud-hosted MongoDB service on Azure.

App_Server, The WeatherExpress project uses the benefits of layer-based design; The app_server layer is responsible to provide data from OpenWeatherMap API and do business over fetched data to provide information for upper layer that is App_API. On this layer, the logic has access to only the database layer and OpenWeatherMap API.

App_API, is responsible to get the request from the APP_Public layer or any 3rd-party app in the future, and pass the request to the downer layer that is APP_Server to provide information for upper layer. On this layer, the logic has access to only the App_Server layer.

App_Cron, is a backend service task that runs every one hour to update the database based on the searched cities and update the weather from the OpenWeatherMap. (The reason to have the Cron Schedule is to prevent getting block because of the repeating calling the OpenWeatherMap API.) On this layer, the logic has access to only the App_Server layer.

App_Public, is responsible to interact with end user to get request and show the information back to the user.
